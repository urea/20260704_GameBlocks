# GameBlocks Project Brief

## Working Assumption

GameBlocks is a game prototype centered on blocks, placement, movement, or transformation. The exact genre and engine are intentionally undecided at scaffold time.

## Early Decisions To Make

1. Target platform: browser, desktop, mobile, or engine-specific build.
2. Core interaction: place blocks, clear blocks, route blocks, combine blocks, or program blocks.
3. Session length: quick puzzle stages, score chase, sandbox, or campaign.
4. Asset style: abstract, toy-like, technical, or character-driven.

## Near-Term Next Steps

1. Write a one-page game loop.
2. Choose the implementation stack.
3. Add the first playable prototype under `src/`.
4. Keep validation runnable after each structural change.
